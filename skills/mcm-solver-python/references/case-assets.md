## 默认使用方式

- 任务开始时先读题目与附件，再读优秀解答。
- 默认把案例作为人类确认前模型准备的一部分。
- 若当前问题与案例明显不相似，降低案例结论权重，但仍可保留论文结构和图表表达层面的参考。
- 用户明确要求参考往年题目或优秀论文时，提高案例参考权重。

## 使用方式

- 默认同时读取题目附件与优秀解答，不要只看其中一侧。
- 优先从优秀解答中提取结构、方法路线和表达方式，不要复制具体文字。
- 若启用 subagent，优先拆为“范例阅读”“模型择优”“合理性校验”三类准备任务。
- 若当前问题只是局部相似，不要把整套案例方法生搬硬套过来。
- 若要生成论文，优先让当前 Python 结果驱动内容，再用案例校正叙述深度与图表形式。

## 避免的问题

- 不要把优秀解答当成标准答案直接改写。
- 不要在没有验证数据与假设一致性的情况下照搬模型。
- 不要把案例中的具体数值、图表结论或参数设置直接用于新题。
