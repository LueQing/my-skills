# 人类确认模型协议

本技能把“向人类确认模型选择”作为必须步骤。无论模型看起来多明显，都先询问，再继续。

## 必问内容

至少向人类确认以下三项：

1. 是否接受推荐模型
2. 是否改用某个备选模型
3. 是否有额外约束、目标或偏好的建模方法需要提前纳入

## 执行规则

- 先输出关键词与模型路由，再发起确认。
- 在人类确认前，不进入 MATLAB 实现、subagent 分工或论文整理。
- 若人类修改模型方向，先重做模型路由，再进入后续步骤。
- 若人类新增约束或目标，把这些内容并入变量、约束和目标函数定义后再继续。

## 推荐提问格式

默认用简洁格式向人类提问：

```markdown
# 模型确认
- 推荐模型：...
- 备选模型：...
- 选择理由：...

请确认：
1. 是否接受推荐模型？
2. 是否改用某个备选模型？
3. 是否有额外约束、目标或方法偏好需要提前纳入？
```

## 避免的问题

- 不要把确认步骤简化成一句模糊的“这样可以吗”。
- 不要只问是否接受推荐模型，而不问备选方案和额外约束。
- 不要在人类尚未回答时默认视为同意。
