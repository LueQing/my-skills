# Subagent 编排协议

当数学建模任务较大时，使用本协议把工作并行拆给多个 subagent。目标是提高吞吐，而不是把问题拆乱。

## 触发原则

- 默认不启用 subagent。
- 仅在以下情况谨慎触发：
- 同时要求题意分析、MATLAB 实现、论文整理
- 题目附件多，数据阅读与代码实现可并行
- 当前题求解还需要参考案例、模板或历史解答
- 用户明确要求“并行”“分工”“subagent”
- 小任务不要拆分。

## 一级默认分工

### 1. 题意/建模 subagent

职责：
- 重述题目目标
- 判断题型
- 提出建模假设
- 梳理变量、参数、约束、评价指标
- 比较 1 到 2 个可行模型并给出推荐

交付：
- 问题拆解
- 推荐模型路线
- 变量和约束清单
- 对 MATLAB 实现的关键提醒

不负责：
- 最终代码落地
- 最终论文定稿
- 最终结论拍板

### 2. MATLAB 实现 subagent

职责：
- 基于已确认模型编写 `.m` 或 `.ipynb`
- 组织数据处理、核心求解、可视化和结果输出
- 说明依赖工具箱、运行方式和未验证点

交付：
- 可运行脚本或 Notebook
- 文件结构建议
- 图表与结果产物清单
- 风险、假设和待确认事项

不负责：
- 擅自改模型方向
- 代替主 agent 输出最终答案

### 3. 论文整理 subagent

职责：
- 在用户需要 LaTeX 或论文交付时，按 `cumcmthesis` 结构组织内容
- 把模型、图表、表格和结果说明整理进论文框架

交付：
- 论文章节框架
- LaTeX 文段草稿
- 图表/表格插入建议
- 需要和 MATLAB 结果核对的清单

不负责：
- 凭空扩写结论
- 脱离计算结果重建模型

## 主 Agent 协议

- 主 agent 先定总方案，再派发并行子任务。
- 主 agent 必须给每个 subagent 明确：
- 任务目标
- 输入材料范围
- 产物边界
- 是否允许继续分工
- 写入范围或输出范围
- 主 agent 最终负责整合、冲突处理和一致性检查。

## 继续分工

- 只有当一级 subagent 的任务本身仍大到明显阻塞时，才继续分工。
- 继续分工必须满足：
- 子任务单一明确
- 边界可一句话说清
- 与同级或上级写入范围不冲突

推荐二级拆分：
- `MATLAB 实现` -> `数据处理` + `核心求解` + `可视化/实验记录`
- `论文整理` -> `结构与摘要` + `图表表格落版`

谨慎拆分：
- `题意/建模` 通常不继续拆；只有题目天然包含多个独立子问题时才拆。

## 避免的问题

- 不要把最终整合权交给 subagent。
- 不要让多个 subagent 同时改同一逻辑块。
- 不要在模型尚未初步确定前，让实现 subagent 盲写 MATLAB。
- 不要让论文 subagent 脱离 MATLAB 实验结果单独产出结论。
- 不要无限递归拆分，导致协调成本高于收益。
